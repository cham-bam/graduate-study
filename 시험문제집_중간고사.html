0강: Introduction

0강은 NLP 소개 슬라이드로, 별도 계산 예제 없음. T/F 문제는 핵심 개념 중심으로 출제.

1강: Language Model Introduction

[Example 1] Language Models for Generation
언어 모델을 이용한 문장 생성
내용: [BOS]부터 시작해서 다음 단어를 반복 샘플링하고, [EOS]가 생성되면 멈춤.
P(w₁) → P(w₂|w₁) → P(w₃|w₁,w₂) → ... → [EOS]
핵심: LM은 조건부 확률의 체인으로 문장을 생성한다. [EOS]는 종료 신호.
[Example 2] p("The cat is on the mat") — N-gram 분해
문장 확률을 Unigram / Bigram / Trigram으로 분해하는 예시
Unigram: p(The) × p(cat) × p(is) × p(on) × p(the) × p(mat)
Bigram: p(The) × p(cat|The) × p(is|cat) × p(on|is) × p(the|on) × p(mat|the)
Trigram: p(The) × p(cat|The) × p(is|The cat) × p(on|cat is) × p(the|is on) × p(mat|on the)
핵심: N이 클수록 더 많은 이전 문맥을 고려하는 Markov 가정.
[Example 3] Vocabulary Size & [UNK]
어휘 크기 설정과 미등록 단어 처리
예: 어휘 크기를 10K로 고정 → 상위 10K 이외의 단어는 훈련 시 모두 [UNK]로 변환
핵심: OOV(Out-of-Vocabulary) 문제를 [UNK] 토큰으로 처리하는 전략.
[Example 4] Numerical Underflow & Log Probability
확률 곱셈의 수치 불안정성과 로그 확률
예: (1/10)¹⁰⁰ = 10⁻¹⁰⁰ → underflow(0으로 반올림) 위험
해결: log(1/10) = -2.3 → 로그 공간에서 덧셈으로 계산
핵심: PPL(Perplexity)은 실제로 log-scale로 계산.

2강: Semantics

[Example 5] Vector Space Basics — Cosine Similarity
4차원 벡터 간의 코사인 유사도 계산
내용: 두 벡터의 dot product(내적) 계산 → 각 벡터의 norm(크기) 계산 → 코사인 유사도 = 내적 / (|A| × |B|)
범위: [-1, 1]. 1에 가까울수록 유사, 0이면 무관계, -1이면 반대 의미.
핵심: 벡터 길이에 영향을 받지 않는 방향 기반 유사도.
[Example 6] PPMI Example
원시 빈도 행렬 → PPMI 가중치 행렬 변환
PMI(w, c) = log₂ [P(w,c) / (P(w)×P(c))]
PPMI: 음수 PMI 값을 모두 0으로 치환
문제점: 희귀 단어에 bias → Power Smoothing(α=0.75) 또는 Add-k smoothing으로 보완
핵심: 두 단어가 우연보다 더 자주 함께 등장하면 높은 PMI.

3강: Word Embeddings

[Example 7] Dense Vector Example
밀집 벡터(Dense Vector)의 각 차원 의미 분산 표현
내용:
- 한 차원: 동물(cat, dog) vs 탈것(car, truck) 구분에 부분 기여
- 다른 차원: 크기(size) 측면 포착
- 또 다른 차원: 격식/감정적 어조 표현
핵심: 어떤 단일 차원도 하나의 개념을 전담하지 않음. 분산 표현(Distributed Representation).
[Example 8] Distributional Hypothesis — "Ong choy"
분포 가설: 문맥이 비슷하면 의미가 비슷하다
예: "Ong choy"의 의미를 모르지만 문맥을 보면:
- "Ong choy is delicious sautéed with garlic / superb over rice / leafy greens"
→ spinach, chard 등과 같은 문맥 → Ong choy = water spinach(공심채)
핵심: "You shall know a word by the company it keeps" (Firth, 1957)
[Example 9] Word2Vec Training Data — "there is a cat on the mat"
Window size=2 기준 Skip-gram 훈련 데이터 생성
입력: "there is a cat on the mat", window=2
훈련 쌍 예시:
(there, is), (there, a)
(cat, is), (cat, a), (cat, on), (cat, the)
핵심: Skip-gram은 중심 단어로 주변 문맥 단어를 예측. 레이블 없이 raw corpus만으로 학습 가능.
[Example 10] Word Analogy — man:woman::king:?
단어 임베딩의 유추 능력
일반식: a:b::c:? → 코사인 유사도를 최대화하는 단어 찾기
벡터 연산: vec(king) - vec(man) + vec(woman) ≈ vec(queen)
핵심: Word2Vec 임베딩 공간에서 의미/문법적 관계가 벡터 방향으로 인코딩됨.
[Example 10-b] Intrinsic Evaluation — WordSim353
내재적 평가 데이터셋: WordSim353
데이터셋: 353개의 단어 쌍 + 인간이 평가한 유사도 점수
예시 쌍: (tiger, cat) → 높은 유사도 / (book, computer) → 낮은 유사도
평가 방법: 임베딩의 코사인 유사도 vs 인간 평가 점수 → Spearman 상관계수
핵심: 내재적 평가(Intrinsic) = 실제 태스크 없이 임베딩 자체의 품질을 단어 유사도·유추로 직접 측정. 외재적 평가(Extrinsic)와 대비되는 개념.
[Example 11] Extrinsic Evaluation — Text Classification & NER
단어 임베딩의 외재적 평가: 분류/개체명 인식
Example 1 (텍스트 분류): 주제/감성 분류. 임베딩 위에 시퀀스 모델링 → 문장 임베딩 → 분류 정확도 측정
Example 2 (NER): 개체명 인식. 단어 임베딩 연결로 span 표현 → Precision/Recall/F1
핵심: 외재적 평가 = 실제 하위 태스크 성능으로 임베딩 품질 판단.

4강: RNNs & Neural Networks

[Example 12] Basic Unit (Perceptron) Example
퍼셉트론 단위 계산 예시
단계:
1. 입력 x (벡터)
2. 가중치 w와 편향 b 적용: z = w·x + b (선형 계산)
3. 비선형 활성화 함수 적용: a = σ(z) (예: tanh, ReLU)
핵심: 선형 연산만 쌓으면 여전히 선형 → 비선형 활성화 함수가 반드시 필요.
[Example 13] 2-layer FFN (Feedforward Network)
2층 피드포워드 신경망 순전파 계산
단계:
1. 입력 x
2. 1층: h = σ(W₁x + b₁) — 비선형 활성화 적용
3. 2층: o = W₂h + b₂
4. 출력: ŷ = softmax(o) → 확률 분포
손실함수: Cross-Entropy (CE) Loss = -log P(정답 클래스)
[Example 14] RNN Limitation — "The bank is on the river"
RNN의 단방향 처리 한계: 단어 중의성 해소 실패
문제: "bank"는 금융/강둑 두 가지 의미 → 뒤에 오는 "river"를 봐야 올바르게 해석 가능
RNN 한계: 왼쪽→오른쪽 처리 → "bank" 처리 시점에 "river"를 아직 보지 못함
핵심: 단방향 RNN은 미래 문맥을 활용할 수 없어 중의성 해소에 취약.

5강: Transformers (시험 범위까지)

[Example 15] Self-Attention Intuition — "The quick brown fox jumps over the lazy dog"
Self-Attention 직관: "jumps"가 어느 단어에 집중하는가
내용: "jumps"에 대한 attention 학습 시
- 높은 attention: "fox"(주어), "over"(전치사)
- 낮은 attention: "the", "lazy"
핵심: Self-Attention은 같은 시퀀스 내 단어들끼리 서로 얼마나 관련 있는지 학습.
[Example 16] Self-Attention Example — weighted sum of context (word "it")
Self-Attention: 문맥 벡터들의 가중 합으로 표현 생성
수식: aᵢ = Σⱼ αᵢⱼ xⱼ, where Σⱼ αᵢⱼ = 1
예: 현재 단어 = "it" → 모든 단어의 표현을 attention 가중치로 합산
αᵢⱼ = Softmax(xᵢ · xⱼ)
핵심: Query(현재 단어) × Key(문맥 단어) = attention score → softmax → 가중치
[Example 17] Self-Attention Computation — Input sequence [1, 2, 3]
QKV 계산 4단계 프로세스
3번 단어의 self-attention 계산 순서:
1. Q, K, V 계산: qᵢ = xᵢWQ, kᵢ = xᵢWK, vᵢ = xᵢWV
2. 3번의 query를 모든 단어의 key와 비교 (dot product)
3. √d로 나눈 후 softmax → attention score (합=1)
4. attention score × value 벡터들을 합산 → contextualized representation
핵심: 출력 = 문맥화된 표현(contextual embedding)
[Example 18] Tokenization — "unhappiness" → "un" + "happy" + "ness"
단어 토큰화의 한계: 서브워드 정보 손실
문제: word-level 토큰화 → "unhappiness" 전체를 하나의 토큰으로 처리 → "un-(접두사)", "-ness(접미사)" 의미 정보 손실
해결: 서브워드 토큰화(Subword Tokenization) — BPE 등
핵심: 형태소 정보가 의미 이해에 중요.
[Example 19] BPE (Byte Pair Encoding) Example
BPE 알고리즘: 빈도 기반 subword 병합
말뭉치: low(5), lowest(2), newer(6), wider(3), new(2)
알고리즘:
1. 초기: 문자 단위로 분리 + end-of-word 기호(_)
2. 가장 빈도 높은 인접 쌍 찾기: "er" (9번) → 병합
3. 다음: "er_" (9번) → 병합
4. 다음: "ne" (8번) → 병합 ...
적용: 병합 규칙 순서대로 test set에 적용 (e.g., "lower_" → "low" + "er_")
[Example 20] BPE Application to Test Set
학습된 BPE 규칙을 테스트 데이터에 적용
병합 규칙: [(e,r), (er,_), (n,e), (ne,w), (l,o), (lo,w), (new,er_), (low,_)]
"newer_" (훈련 데이터에 있음): → "newer_" 전체 단어로 토큰화
"lower_" (훈련 데이터에 없는 단어): → "low" + "er_" 로 분리
핵심: BPE는 OOV 없이 서브워드 단위로 분해 가능.

단어-단어 공기 행렬 구성: "Taylor releases a new pop album..."
예: 입력 문장 "Taylor releases a new pop album with a hit song", window=2
공기 쌍:

Transformers Details PDF (시험 범위까지)

[Example 22] Self-attention 4-step Illustration (3-word sequence)
3단어 시퀀스의 Self-attention 4단계 시각화
[1단계] Q, K, V 계산 (학습된 행렬 W_Q, W_K, W_V 적용)
[2단계] 3번 단어의 query를 모든 단어의 key와 비교
[3단계] softmax로 attention score 도출 (합=1)
[4단계] weighted sum of value vectors → 출력: [o₁, o₂, o₃] contextualized representation
[Example 23] Stacking Self-Attention Layers — "picnic on bank/river"
Self-Attention 레이어 쌓기: 문맥 정보 전파
1층만 있을 때 문제: "picnic"과 "bank"는 static embedding에서 유사하지 않음 → attention 가중치가 낮게 할당
레이어 쌓기의 효과: "picnic" ↔ "river" 유사 / "river" ↔ "bank" 유사 → 2층에서 "picnic"과 "bank"의 관계가 전파됨
핵심: 레이어를 쌓을수록 간접적 문맥 관계도 포착 가능.
[Example 24] BERT: Stacking Transformer Layers
BERT: 마지막 레이어 출력 = contextual embedding
BERT 구조: 여러 Transformer 레이어(self-attention + FFN) 쌓기
핵심 결과: "The outputs from the last layer become contextual embeddings"
→ 각 단어의 최종 출력 벡터는 문맥을 반영한 밀집 표현
vs. Word2Vec: Word2Vec은 고정 임베딩 / BERT는 문맥에 따라 달라지는 임베딩

T/F
Q1. Bigram LM에서 각 단어의 확률은 바로 직전 단어에만 의존한다.
TRUE — Bigram: P(wₙ|wₙ₋₁)
Q2. Trigram LM에서 p("The cat is on the mat")을 계산하려면 6개의 조건부 확률을 곱해야 한다.
TRUE — p(The)×p(cat|The)×p(is|The cat)×p(on|cat is)×p(the|is on)×p(mat|on the)

FALSE — N이 클수록 해당 N-gram이 훈련 데이터에 등장하지 않을 가능성이 높아짐 → 희소성 증가.
Q4. [BOS]와 [EOS]는 각각 문장의 시작과 끝을 나타내는 특수 토큰이다.
TRUE
Q5. N-gram 확률은 MLE(최대우도추정)로 훈련 말뭉치의 빈도를 이용해 추정한다.
TRUE — P(wₙ|wₙ₋₁) = count(wₙ₋₁, wₙ) / count(wₙ₋₁)
Q6. Perplexity(PPL)가 낮을수록 언어 모델의 성능이 더 좋다.
TRUE — PPL = 평균적으로 얼마나 당황(perplexed)하는지의 척도. 낮을수록 예측이 잘 됨.
Q7. 확률값들을 로그 공간에서 덧셈으로 계산하면 수치적 언더플로(underflow)를 방지할 수 있다.
TRUE — log(a×b) = log(a) + log(b)
Q8. [UNK] 토큰을 사용하면 OOV(Out-of-Vocabulary) 문제를 완전히 해결할 수 있다.
FALSE — OOV를 처리하지만, [UNK]로 대체하면 해당 단어의 고유 의미 정보는 손실됨.
Q9. Unigram LM은 단어 간의 상관관계를 포착하지 못한다.
TRUE — Unigram은 각 단어를 독립적으로 처리: P(w₁,...,wₙ) = ΠP(wᵢ)
Q10. 언어 모델은 문장 생성 외에도 다음 단어 확률 예측, 문장 점수화 등 다양한 용도로 활용된다.
TRUE

2강: Semantics — T/F 10문제

T/F
Q1. Cosine Similarity의 범위는 [0, 1]이다.
FALSE — 범위는 [-1, 1]. 음수 PMI는 가능하지만 PPMI에서 0으로 처리.
Q2. Cosine Similarity는 벡터의 크기(magnitude)에 영향을 받지 않는다.
TRUE — 방향(각도)만 비교하기 때문.
Q3. PPMI는 PMI의 음수값을 0으로 교체한 것이다.
TRUE — Positive PMI: max(PMI, 0)
Q4. PMI가 높을수록 두 단어가 우연보다 더 자주 함께 등장한다는 의미이다.
TRUE — PMI = log[P(w,c) / P(w)P(c)]. 양수면 우연 이상의 연관성.
Q5. PPMI는 빈도가 낮은 단어에 대해 과소추정(underestimation) 경향이 있다.
FALSE — 반대로 희귀 단어에 PMI가 과대추정(bias toward infrequent)되는 경향. Power Smoothing으로 보완.
Q6. TF-IDF는 단어 쌍의 연관성을 단어 수준(pair level)에서 측정한다.
FALSE — TF-IDF는 문서 수준(document level). PMI가 단어 쌍 수준.
Q7. 벡터 공간 모델(Vector Space Model)에서 유사한 의미의 단어는 유사한 벡터 표현을 가진다.
TRUE — 분포 가설(Distributional Hypothesis) 기반.
Q8. One-hot 벡터에서는 모든 단어 쌍의 코사인 유사도가 0이다.
TRUE — 각 단어가 서로 다른 차원을 가지므로 내적=0 → cos=0.
Q9. PPMI Power Smoothing에서 α < 1이면 희귀 단어의 확률이 증가한다.
TRUE — P(c)^α (α=0.75)로 분모를 줄여 희귀 단어 확률을 boost.
Q10. TF-IDF에서 IDF 값이 높다는 것은 해당 단어가 많은 문서에 걸쳐 자주 등장한다는 의미이다.
FALSE — IDF = log(N/df). IDF가 높으면 특정 문서에만 등장(희귀). 많은 문서에 등장하면 IDF가 낮음.

PART 3: 단답형 예상문제 (Example 기반, 풀이 단계 포함)

1강 — 단답형

SA-1. 단답형
문장 "The cat is on the mat"의 확률을 Bigram LM으로 분해하시오. (단, [BOS]와 [EOS]는 생략)
[풀이 단계]
① Bigram 공식: P(wₙ|wₙ₋₁) = count(wₙ₋₁,wₙ) / count(wₙ₋₁)
② P("The cat is on the mat")
= P(The) × P(cat|The) × P(is|cat) × P(on|is) × P(the|on) × P(mat|the)
[정답] 위 6개 항의 곱
SA-2. 단답형
확률값들의 곱 (1/10)¹⁰⁰을 직접 계산하면 어떤 수치적 문제가 발생하는가? 그리고 이를 어떻게 해결하는가?
[풀이 단계]
① 문제: (1/10)¹⁰⁰ = 10⁻¹⁰⁰ → 컴퓨터가 0으로 반올림 = Underflow
② 해결: 로그(log) 변환 사용
③ log(1/10) = -2.3 → 곱셈 대신 덧셈으로 계산
④ log(a × b) = log(a) + log(b)
[정답] Underflow 문제 / Log-scale로 덧셈 연산

2강 — 단답형

SA-3. 단답형
두 벡터 a = [1, 0, 1, 0], b = [0, 1, 1, 0]의 코사인 유사도를 구하시오.
[풀이 단계]
① 내적(dot product): a·b = 1×0 + 0×1 + 1×1 + 0×0 = 1
② |a| = √(1+0+1+0) = √2
③ |b| = √(0+1+1+0) = √2
④ cos(a,b) = 1 / (√2 × √2) = 1/2 = 0.5
[정답] 0.5
SA-4. 단답형
PPMI에서 Power Smoothing(α=0.75)을 적용하는 이유와 방법을 설명하시오.
[풀이 단계]
① 문제: 희귀 단어가 PMI에서 과대평가(bias toward infrequent)
② 해결: 분모의 문맥 단어 확률에 power α를 적용
③ PPMI_α(w,c) = max(log[P(w,c) / (P(w) × P(c)^α)], 0)
④ α < 1 → 희귀 단어의 P(c) 값이 상대적으로 커짐 → PMI 값이 낮아져 보정됨
[정답] 희귀 단어의 bias 보정 / 분모 P(c)를 P(c)^α로 대체

3강 — 단답형

SA-5. 단답형
입력 문장 "there is a cat on the mat"에서 window size=2일 때, 단어 "cat"의 훈련 쌍(word-context pairs)을 모두 쓰시오.
[풀이 단계]
① 문장: there(1) is(2) a(3) cat(4) on(5) the(6) mat(7)
② window=2: cat 기준 왼쪽 2개, 오른쪽 2개
③ 왼쪽: is(2), a(3) / 오른쪽: on(5), the(6)
[정답] (cat, is), (cat, a), (cat, on), (cat, the)
SA-6. 단답형
Word2Vec analogy에서 "man:woman::king:?"을 벡터 연산으로 구하는 방법을 기술하시오.
[풀이 단계]
① 일반 공식: a:b::c:? → vec(b) - vec(a) + vec(c)와 가장 유사한 단어
② 대입: vec(woman) - vec(man) + vec(king)
③ 코사인 유사도를 최대화하는 단어를 어휘에서 탐색
[정답] argmax_{w} cos(vec(w), vec(woman)-vec(man)+vec(king)) → queen

4강 — 단답형

SA-7. 단답형
2-layer FFN에서 입력 x가 주어질 때 출력 확률 분포 ŷ를 구하는 순전파(forward pass) 단계를 순서대로 기술하시오.
[풀이 단계]
① 1층 선형 변환: z₁ = W₁x + b₁
② 1층 비선형 활성화: h = σ(z₁) (예: ReLU, tanh)
③ 2층 선형 변환: z₂ = W₂h + b₂
④ 출력 확률화: ŷ = softmax(z₂)
[정답] z₁ → h=σ(z₁) → z₂ → ŷ=softmax(z₂)
SA-8. 단답형
단방향 RNN이 "The bank is on the river"에서 "bank"의 의미를 정확히 파악하지 못하는 이유를 설명하시오.
[풀이 단계]
① 단방향 RNN은 왼쪽→오른쪽으로 처리
② "bank" 처리 시점: The, bank까지만 본 상태
③ "river"는 "bank" 뒤에 위치 → 아직 읽지 않음
④ "bank"의 의미(금융 vs 강둑)는 "river"로 해소 가능하지만 접근 불가
[정답] 단방향 처리로 미래 문맥("river")에 접근 불가 → 중의성 해소 실패

5강 — 단답형

SA-9. 단답형
3단어 시퀀스 [x₁, x₂, x₃]에서 x₃의 self-attention을 계산하는 4단계를 순서대로 기술하시오.
[풀이 단계]
① Q, K, V 계산: q₃=x₃WQ, k₁=x₁WK, k₂=x₂WK, k₃=x₃WK, v₁~v₃ 계산
② Attention score: q₃와 k₁, k₂, k₃ 각각 dot product → √d_k로 나눔
③ Softmax: score들을 softmax → α₃₁, α₃₂, α₃₃ (합=1)
④ Weighted sum: o₃ = α₃₁v₁ + α₃₂v₂ + α₃₃v₃ → contextual representation
[정답] ①QKV 계산 → ②dot product/√d → ③softmax → ④weighted sum
SA-10. 단답형
코퍼스: low(5회), lowest(2회), newer(6회), wider(3회), new(2회)
BPE 첫 번째 병합 단계에서 어떤 쌍이 선택되고 병합 결과는 어떻게 되는가?
[풀이 단계]
① 초기화: 모든 단어를 문자 단위로 분리 (+ end-of-word _ 추가)
② l o w _ / l o w e s t _ / n e w e r _ / w i d e r _ / n e w _
③ 모든 인접 쌍의 빈도 카운트:
   (e, r): newer(6) + wider(3) = 9번 → 최고 빈도
④ "e"와 "r"을 병합 → "er"
[정답] 선택 쌍: (e, r), 결과: "er" (9번 등장)
SA-11. 단답형
BPE 병합 규칙이 [(e,r), (er,_), (n,e), (ne,w), (l,o), (lo,w), (new,er_), (low,_)]로 학습되었을 때, 테스트 단어 "lower_"는 어떻게 토큰화되는가? 단계별로 설명하시오.
[풀이 단계]
① 초기: l o w e r _
② 규칙1 (e,r) 적용: l o w er _
③ 규칙2 (er,_) 적용: l o w er_
④ 규칙5 (l,o) 적용: lo w er_
⑤ 규칙6 (lo,w) 적용: low er_
⑥ (new,er_) 규칙은 해당 없음 (low≠new)

SA-12. 단답형
Self-Attention 레이어를 여러 층 쌓는 이유를 "picnic on bank" 예시를 들어 설명하시오.
[풀이 단계]
① 1층만 있을 때: attention score = static embedding의 유사도에 의존
② "picnic"과 "bank"는 static embedding에서 유사하지 않음 → 낮은 attention
③ 하지만: "picnic" ↔ "river" 유사, "river" ↔ "bank" 유사
④ 레이어 2에서: 1층 출력을 입력으로 사용 → "picnic"이 "river"를 통해 "bank"와 연결
[정답] 간접적 문맥 관계를 여러 층을 통해 전파 가능
SA-13. 단답형
Word2Vec(정적 임베딩)과 BERT(문맥적 임베딩)의 차이를 "bank(은행/강둑)" 단어를 예시로 설명하시오.

② "river bank"든 "bank account"든 같은 벡터 → 중의성 처리 불가
