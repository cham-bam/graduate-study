0강 — Introduction to NLP
NLP란?
p.5
NLP 정의
머신러닝 + 언어학의 융합 분야.
목표: 컴퓨터가 인간 언어를 이해·해석·생성할 수 있게 함.
핵심 문제 = 언어 모델링 (Language Modeling)
p.14~16
언어 모델의 핵심 역할 시험
언어 모델 = 단어 시퀀스에 확률을 부여하는 모델
  • 텍스트 이해: p("The cat is on the mat") >> p("Truck the earth on")
  • 텍스트 생성: p(w | "The cat is on the") → "mat"
P(w₁, w₂, ..., wₙ) = P(w₁) × P(w₂|w₁) × ... × P(wₙ|w₁,...,wₙ₋₁)
모든 NLP 태스크는 언어 모델링 문제로 변환 가능!

NLP 발전 단계 시험
시기방법론대표 기술

1강 — N-gram Language Models
언어 모델 기초
p.6~9
Autoregressive 언어 모델 시험
Autoregressive 가정: 각 단어의 확률은 이전 토큰들에만 의존.
P(w₁,...,wₙ) = ∏ P(wₜ | w₁,...,wₜ₋₁)
생성 방법: P(w | 이전 단어들)에서 반복 샘플링 → [EOS] 나올 때까지.
BOS(시작 토큰)으로 시작, EOS(끝 토큰)로 종료
N-gram 핵심
p.16~17
N-gram 가정 시험
전체 이전 맥락을 추적하기 어려움 → 이전 N-1개 단어만 봄.
종류의존 단어 수예시
Unigram (N=1)없음 (독립)P(mat)
Bigram (N=2)이전 1개P(mat|the)
Trigram (N=3)이전 2개P(mat|on the)
p.18
N-gram 학습 (MLE) 시험
빈도(counts)로 확률 추정 = 최대우도추정 (MLE):
P(wₙ | wₙ₋₁) = count(wₙ₋₁, wₙ) / count(wₙ₋₁)
코퍼스에서 등장 횟수를 세서 확률 계산.
p.25
N-gram 특성 & 트레이드오프 시험
N이 클수록:
  • ✅ 더 많은 문맥 반영 (단어 상관관계 포착)
  • ❌ Sparsity 증가 (어휘 크기의 N제곱만큼 파라미터 필요)
어휘 크기 10K 기준: Unigram=10K, Bigram=100M, Trigram=1T 파라미터
Sparsity = 학습 데이터에 없는 N-gram → 확률 0 → 문장 전체 확률 0!
Smoothing
p.29~36
Smoothing 기법 시험
목적: 등장하지 않은 N-gram에도 0이 아닌 확률 부여.
방법아이디어문제점
Add-one (Laplace)모든 카운트에 +1과도한 평탄화 (over-smoothing)
Add-kk < 1을 더함k는 validation set으로 튜닝
Interpolation여러 N-gram 확률 혼합가중치 λ 튜닝 필요
Backoff고차 N-gram 없으면 저차로 후퇴discount 계수 필요
Add-one: P(wₙ|wₙ₋₁) = (count(wₙ₋₁,wₙ) + 1) / (count(wₙ₋₁) + V)
p.37
OOV (Out-of-Vocabulary) 처리
학습 데이터에 없는 단어 → [UNK] 토큰으로 대체.
학습 시: 드문 단어를 [UNK]로 교체 후 확률 추정.
테스트 시: 미등장 단어 → [UNK] 확률 사용.
언어 모델 평가
p.42~45
Perplexity (PPL) 시험
언어 모델의 내재적 평가 지표.
"다음 단어에 얼마나 당황하는가?" — 낮을수록 좋음.
PPL = P(w₁,...,wₙ)^(-1/N)
log PPL = -1/N × Σ log P(wₜ | w₁,...,wₜ₋₁) (수치 안정성을 위해 log scale)
구분설명예시
Intrinsic언어 모델 자체 품질 측정Perplexity
Extrinsic실제 태스크 성능 측정분류 정확도, 번역 BLEU
2강 — Word Senses & Semantics
Word Senses
p.31~36
어휘 의미 관계 유형 시험
관계정의예시
Synonyms (동의어)같은 의미happy ≈ joyful
Antonyms (반의어)반대 의미hot ↔ cold
Hyponym (하위어)더 구체적인 개념rose → flower
Hypernym (상위어)더 일반적인 개념flower ← rose
Polysemy (다의어)한 단어, 여러 관련 의미bank: 은행/강둑
Word Sense Disambiguation (WSD): 문맥에 맞는 의미 판별 태스크
p.40~41
Connotation (함의)
단어의 글자 그대로의 의미를 넘어선 정서적·문화적 연상.
3차원으로 표현:
  • Valence: 유쾌함 정도 (happy↑, unhappy↓)
  • Arousal: 감정 강도 (excited↑, calm↓)
  • Dominance: 통제감 (controlling↑, influenced↓)
최초의 다차원 단어 벡터 표현 아이디어!
WordNet
p.43~48
WordNet
수동으로 구축된 대형 어휘 데이터베이스.
Synset: 같은 의미를 가진 단어들의 집합.
단어 관계(동의어, 반의어, 상/하위어) 정의.
한계: 구축/유지 비용↑, 도메인 특화 용어 부족, 문맥 고려 불가.
→ 이 한계를 극복하려고 벡터 표현 (Word Embeddings) 등장!
Vector Space Model
p.51~54
벡터 공간 기초 & 코사인 유사도 시험
단어를 다차원 벡터로 표현. 의미가 유사한 단어는 공간에서 가까움.
cosine(u, v) = (u·v) / (|u| × |v|) → 범위: [-1, 1]
코사인 유사도 특성:
  • 벡터 길이에 영향받지 않음
  • 대칭: sim(u,v) = sim(v,u)
  • 직관적 기하학적 해석 가능
p.55~58
One-hot 벡터 & 문서 표현
가장 단순한 단어 벡터: one-hot (해당 단어 인덱스만 1, 나머지 0).
단어-문서 행렬: 단어가 각 문서에 등장하는 빈도로 표현.
단어-단어 행렬: ±4 단어 윈도우 내 동시 등장 횟수로 표현.
TF-IDF & PMI
p.24~30
TF-IDF & PMI 시험
단순 빈도 대신 가중치를 부여하는 방법:
방법아이디어수식
TF빈도를 log 압축log(count + 1)
IDF드문 단어에 높은 가중치log(N/df)
TF-IDFTF × IDF문서 내 중요도
PMI우연보다 얼마나 더 함께 등장?log P(x,y)/(P(x)P(y))
PPMIPMI의 음수값 → 0으로 대체max(PMI, 0)
3강 — Word Embeddings
Sparse vs Dense 벡터
p.33~36
Sparse vs Dense 벡터 비교 시험
Sparse (희소)Dense (밀집)
차원어휘 크기 (>10K)100~300 (훨씬 작음)
값대부분 0대부분 비영(실수)
해석각 차원 = 특정 단어/문서각 차원 = 추상적 의미
예시One-hot, TF-IDFWord2Vec, GloVe
Dense 장점: 컴팩트, 일반화 좋음, 신경망 입력으로 효율적.
Dense 단점: 해석 어려움 (각 차원 의미 불명확).
Distributional Hypothesis
p.38
Distributional Hypothesis 시험
"비슷한 문맥에 등장하는 단어는 비슷한 의미를 가진다"
"You shall know a word by the company it keeps"
Word Embedding의 핵심 이론적 근거.
예: "Ong choy"를 몰라도 "garlic", "rice", "salty leaves"와 자주 등장 → 채소류임을 유추
Word2Vec
p.44~51
Word2Vec (Skip-gram) 시험
목표: 중심 단어로 주변 단어(context)를 예측하도록 학습.
각 단어는 두 벡터를 가짐: 중심 단어 벡터(u), 문맥 단어 벡터(v).
P(o | c) = exp(uₒ · vc) / Σw exp(uw · vc)
Negative Sampling: 전체 어휘 합산 대신 랜덤 샘플링으로 근사.
→ 실제 쌍(positive) vs 랜덤 쌍(negative)을 이진 분류로 학습.
CBOW = 문맥 단어들로 중심 단어 예측 (Skip-gram의 반대)
p.45~46
Word2Vec 설정 & 학습 데이터 시험
학습 데이터: (중심 단어, 문맥 단어) 쌍 — 사람이 레이블링 불필요!
"there is a cat on the mat" + window=2 → (cat, is), (cat, a), (cat, on), (cat, the) 등
하이퍼파라미터:
  • 임베딩 차원: 보통 100~300
  • Context window: 보통 5~10 (작으면 문법적, 크면 의미적 정보)
  • Negative sample 수: 보통 5~10
p.52
SGD 최적화 시험
θ ← θ + α × ∇θ J(θ)
파라미터를 gradient 방향으로 업데이트.
α = learning rate (보통 0.02~0.05).
임베딩 평가
p.55~59
Word Embedding 평가
Intrinsic (내재적):
  • Word Similarity: 코사인 유사도와 인간 판단 상관관계
  • Word Analogy: "man:woman = king:?" → 벡터 연산으로 답 찾기
analogy: vking - vman + vwoman ≈ vqueen
Extrinsic (외재적): 텍스트 분류, NER 등 실제 태스크 성능.
4강 — Recurrent Neural Networks
신경망 기초
p.3
Word Similarity (복습)
단어 임베딩 간 코사인 유사도로 의미 유사도 측정. 높을수록 의미가 가까움.
4강은 3강(임베딩) 바로 다음 → 임베딩이 입력으로 쓰임
p.4
Sequence Modeling 목표
텍스트 시퀀스를 이해·처리·생성하는 것.
  • 문맥 의존 표현 학습
  • 장거리 의존성 포착
  • 복잡한 단어 관계 처리
DNN은 언어의 계층 구조(글자→단어→문장)를 층별로 모델링함
p.7
퍼셉트론 (Perceptron) — 기본 단위
신경망의 최소 단위. 계산 순서:
  1. 입력 x
  2. 선형 계산: z = Wx + b
  3. 비선형 활성화: y = f(z)
z = w₁x₁ + w₂x₂ + ... + b → y = sigmoid(z)
p.8
퍼셉트론 계산 예시
구체적인 숫자로 퍼셉트론 계산 과정 보여주는 슬라이드.
W(가중치)와 b(바이어스)가 학습되는 파라미터
p.9
비선형 활성화 함수
왜 비선형? → 선형을 아무리 쌓아도 선형 → 복잡한 패턴 학습 불가.
함수범위특징
Sigmoid0 ~ 1확률 출력, 초기에 많이 씀
tanh-1 ~ 10 중심, LSTM에서 사용
ReLU0 ~ ∞단순, 현재 가장 많이 씀
FFN & 학습
p.11
Feedforward Network (FFN)
FFN = MLP = 여러 퍼셉트론을 층으로 쌓은 것.
각 층: 가중치 행렬(W) + 바이어스 벡터(b).
아래 층 출력 → 위 층 입력으로 전달 (방향이 한쪽으로만).
p.12
2-layer FFN 계산 예시
h = f(W₁x + b₁) ← 1층: 비선형 활성화
z = W₂h + b₂ ← 2층: 선형
ŷ = softmax(z) ← 확률 분포 출력
1층 = 특징 추출, 2층 = 분류
p.13
훈련 목적함수 — Cross-Entropy Loss 시험
정답과 예측의 차이를 수치화. 분류 문제의 표준 손실함수.
CE Loss = -log P(정답 클래스)
→ 정답 클래스에 높은 확률을 줄수록 손실 감소.
NLL Loss(Negative Log Likelihood)와 동일한 개념.
정답은 one-hot 벡터 → 정답 클래스 하나의 확률만 보면 됨
p.14
모델 학습 — Forward Pass
경사하강법(Gradient Descent)으로 파라미터 최적화.
  1. 파라미터 랜덤 초기화
  2. Forward: 입력 → 모델 → 손실값 계산 (예측이 얼마나 틀렸나?)
  3. Backward: gradient 계산 후 파라미터 업데이트
p.15
모델 학습 — Backward Pass
손실값 → 역방향으로 gradient 계산 → 파라미터 수정.
θ ← θ - α × ∇L(θ)
Chain Rule로 gradient 계산. PyTorch가 자동으로 처리 (autograd).
Simple Neural Language Model
p.17
Simple Neural Language Model
FFN을 언어 모델로 사용.
  • 입력: 이전 N개 단어의 임베딩 (concatenate)
  • Hidden layer 통과
  • 출력: 다음 단어 확률 분포
p.18
Neural LM의 장점
  • Sparsity 해결: 모든 단어에 0보다 큰 확률 부여 (N-gram은 미등장 시 0)
  • 의미 유사성 반영: 비슷한 의미 단어 → 비슷한 확률 분포
p.19
Simple Neural LM의 한계 시험
컨텍스트 창이 고정됨 → N-gram과 같은 문제.
N을 늘리면 볼 수 있는 과거는 많아지지만, 파라미터도 그만큼 증가.
→ 이 문제를 해결하려고 RNN 등장!
RNN 핵심
p.21
RNN 개요 시험
임의 길이의 입력을 처리할 수 있는 신경망.
핵심 아이디어: 모든 단어에 같은 가중치(W)를 재사용.
"같은 뇌로 단어를 하나씩 순서대로 읽는" 구조
p.22
RNN 계산 (개념) 시험
각 시점(t)의 hidden state 계산:
hₜ = f(Wₕ · hₜ₋₁ + Wₓ · xₜ + b)
  • hₜ₋₁: 이전 시점의 기억 (과거 정보)
  • xₜ: 현재 단어 임베딩 (현재 정보)
  • Wₕ, Wₓ: 모든 시점에서 공유되는 파라미터
p.23
RNN 계산 절차
  1. [BOS] 토큰으로 시작, h₀ 초기화
  2. 각 단계: hₜ = f(Wₕhₜ₋₁ + Wₓxₜ)
  3. 출력: ŷₜ = softmax(Wₒhₜ)
  4. 반복 (W는 항상 같은 것 사용)
p.24
RNN Weight Tying
출력층 행렬 = 입력 임베딩 행렬 (같은 걸 공유).
  • 파라미터 수 감소
  • 학습 효율 & 성능 향상
차원이 같으니까 가능. 입출력이 같은 어휘 공간에 있기 때문
p.25
RNN으로 언어 모델링
P(wₜ | w₁, ..., wₜ₋₁) 계산이 목표.
t-1 시점의 hₜ₋₁이 이전 모든 단어를 압축하여 저장.
이론상 무한 과거를 기억할 수 있지만 실제로는 어려움 (뒤에서 설명)
p.26
RNN 언어 모델 학습 (Teacher Forcing)
매 시점마다 "다음 단어 맞추기" 형태로 학습.
훈련 시: 항상 정답 단어를 다음 입력으로 줌 = Teacher Forcing.
손실 = 각 시점 Cross-Entropy의 합.
p.27
RNN 텍스트 생성
  1. [BOS] 입력
  2. softmax에서 단어 샘플링
  3. 그 단어를 다음 입력으로
  4. [EOS]가 나올 때까지 반복
생성 시엔 모델 자신의 출력을 다음 입력으로 사용 (훈련과 다름!)
RNN 한계
p.29
Vanishing & Exploding Gradient 시험
역전파 시 gradient를 시간 역방향으로 계속 곱함.
  • Vanishing: 작은 값 반복 곱 → 0으로 수렴 → 먼 과거 학습 불가
  • Exploding: 큰 값 반복 곱 → 무한대 → 학습 발산
Gradient Clipping으로 exploding은 어느정도 완화 가능
p.30
장거리 의존성 포착 어려움 시험
hidden state 크기는 고정인데 문장이 길어지면 → 초반 정보가 덮어씌워짐.
최근 단어에 더 많은 가중치 → 문장 앞부분을 "잊어버림".
이론 ≠ 현실: 이론상 무한 기억, 실제는 최근 것만
p.31
양방향성 부재 시험
RNN은 왼쪽 → 오른쪽으로만 처리.
예: "The bank is on the river"
→ "bank" 처리 시점에 "river"가 아직 안 나옴 → 의미 파악 불가.
Bidirectional RNN으로 해결 (p.41)
p.32
Exposure Bias 시험
  • 훈련 시: 항상 정답 단어를 다음 입력으로 줌 (Teacher Forcing)
  • 생성 시: 모델 자신의 예측을 다음 입력으로 사용
→ 한 번 틀리면 틀린 것 기반으로 계속 예측 → 오류 누적.
훈련과 실제 상황의 불일치가 문제.
LSTM & Advanced RNNs
p.34
LSTM 개요 시험
RNN 문제: hidden state가 금방 덮어씌워짐.
LSTM 해결책: cell state(장기 메모리) 추가 + 3개의 게이트로 정보 흐름 제어.
게이트역할
Forget gate뭘 잊을지
Input gate뭘 기억할지
Output gate뭘 출력할지
입력: 현재 단어 임베딩 + 이전 hidden state + cell state
p.35
LSTM — Forget Gate
fₜ = σ(Wf · [hₜ₋₁, xₜ] + bf)
sigmoid → 0~1 사이 값.
0이면 "완전히 잊어", 1이면 "전부 기억".
cell state × fₜ → 일부 정보 삭제.
p.36
LSTM — Input Gate
iₜ = σ(Wi · [hₜ₋₁, xₜ] + bi)
새로운 정보 중 얼마나 cell state에 넣을지 결정하는 밸브.
0~1 사이 값.
p.37
LSTM — Candidate Cell State
c̃ₜ = tanh(Wc · [hₜ₋₁, xₜ] + bc)
tanh → -1~1 사이 값.
"현재 입력에서 뽑아낸 새 정보 후보".
Input gate와 곱해져서 실제 반영량 결정.
p.38
LSTM — Cell State 업데이트 시험
cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ c̃ₜ
= "지울 건 지우고 (forget gate)" + "추가할 건 추가 (input gate × candidate)"
이게 LSTM의 핵심! 장기 기억 유지의 비결
p.39
LSTM — Output Gate
oₜ = σ(Wo · [hₜ₋₁, xₜ] + bo)
cell state 전체를 다 내보내지 않고 필요한 부분만 선택.
p.40
LSTM — Hidden State 업데이트
hₜ = oₜ ⊙ tanh(cₜ)
cell state를 tanh로 압축 → output gate로 필터링 → hidden state.
이 hₜ가 다음 시점 or 상위 레이어로 전달됨.
p.41
Bidirectional RNN 시험
앞→뒤 RNN + 뒤→앞 RNN 두 개를 따로 학습.
각 단어 위치에서 두 hidden state를 concatenate.
→ 앞뒤 문맥 모두 반영 가능.
"bank" 문제 해결: 나중에 나오는 "river"도 참고 가능
p.42
Deep RNN
RNN 레이어를 여러 층 쌓음.
아래 층 출력 → 위 층 입력.
층이 깊어질수록 더 추상적인 표현 학습.
마지막 층 출력 = 최종 표현.
5강 — Transformer Language Models
Transformer 개요 & 동기
p.3
Transformer 개요
Transformer = attention으로 RNN의 반복 연산을 대체한 DNN 구조.
현재 거의 모든 LLM(GPT, BERT 등)의 기반.
2017년 "Attention is All You Need" 논문에서 등장
p.4
Transformer vs. RNN
  • RNN: 순서대로 하나씩 처리 (recurrent)
  • Transformer: attention으로 전체 동시에 처리
p.5
Transformer 등장 이유 (RNN 3가지 한계 해결) 시험
문제RNNTransformer
병렬 처리한 번에 하나씩전체 동시 처리
장거리 의존성vanishing gradient어디든 직접 접근
양방향성단방향기본이 양방향
p.6
Transformer 레이어 구성 시험
각 Transformer 레이어 = 아래 4개로 구성:
  1. Self-Attention
  2. Feedforward Network (FFN)
  3. Residual Connection
  4. Layer Normalization
이 블록을 N번 쌓아서 깊게 만듦.
Self-Attention
p.7
Self-Attention 직관 시험
"이 단어를 이해하려면 다른 어떤 단어에 집중해야 하나?"
각 단어가 같은 시퀀스 내 다른 단어들을 참조.
예: "jumps" → "fox"(주어), "over"(전치사)에 높은 attention / "the", "lazy"에는 낮은 attention.
p.8
Self-Attention 예시 (it)
"it"의 표현 = 문장 내 모든 단어 표현의 가중합.
aᵢ = Σ αᵢⱼ × xⱼ , Σαᵢⱼ = 1
αᵢⱼ = attention score (softmax 후 합 = 1).
p.9
Attention Score 계산 & Q/K 분리 이유 시험
αᵢⱼ = Softmax(xᵢ · xⱼ)
왜 같은 벡터 두 개를 내적하지 않나?
→ 자기 자신과의 내적이 항상 최대 → 항상 자기한테만 집중하는 문제 발생.
→ 그래서 Query와 Key를 다른 행렬로 분리!
Q, K, V
p.10
Query, Key, Value 개념 시험
단어마다 3가지 역할 벡터:
벡터역할비유
Q (Query)어떤 정보를 찾나?검색 쿼리
K (Key)어떤 정보를 가졌나?검색 인덱스
V (Value)실제 전달할 내용검색 결과
Q와 K로 attention score 계산 → 그 score로 V를 가중합.
p.11
Q, K, V 생성 방법
입력 단어 벡터 x에 3개의 가중치 행렬을 곱해서 생성:
q = xWQ , k = xWK , v = xWV
WQ, WK, WV는 학습되는 파라미터 행렬.
p.12
Self-Attention 전체 계산 시험
αᵢⱼ = Softmax(qᵢ · kⱼ / √d)
aᵢ = Σ αᵢⱼ × vⱼ
√d로 나누는 이유: 차원(d)이 클수록 내적값이 커져 softmax가 극단적으로 쏠림 → √d로 나눠서 안정화.
p.13
Self-Attention 시각화 (3단어 예시)
x1, x2, x3 → Q,K,V 계산 → x3의 Q를 모든 K와 비교 → softmax → V 가중합 → 출력.
그림 보면서 흐름 익히기: Q와 K 내적 → √d 나누기 → softmax → V 가중합
Multi-Head & 병렬 계산
p.14
Multi-Head Self-Attention 시험
attention head를 여러 개 사용.
각 head가 서로 다른 종류의 관계를 학습 (문법적, 의미적, 위치적 등).
각 head 결과를 concatenate → 선형 변환 → 최종 출력.
p.15
QKV 병렬 계산
토큰마다 QKV 계산이 독립적 → 행렬 연산으로 전체 동시 처리.
Q = XWQ , K = XWK , V = XWV
X = 전체 입력 행렬 (모든 토큰 포함). GPU 행렬 연산으로 빠름.
p.16
Attention 행렬 연산
A = Softmax(QKᵀ / √d) × V
행렬 한 번으로 모든 단어의 attention을 동시에 계산.
Attention matrix = 각 단어 쌍의 attention score 행렬.
양방향 vs 단방향
p.17
Bidirectional Self-Attention (Encoder) 시험
모든 위치가 모든 위치에 attention 가능 → 양방향.
이런 구조 = Transformer Encoder.
대표 모델: BERT.
용도: 텍스트 분류, NER 등 이해(NLU) 태스크.
p.18
Unidirectional Self-Attention (Decoder) 시험
현재 위치는 이전 위치들만 볼 수 있음 → 단방향(Causal).
미래 토큰은 -∞ 마스킹으로 차단.
이런 구조 = Transformer Decoder.
대표 모델: GPT.
용도: 텍스트 생성(NLG).
Position Encoding
p.19
Position Encoding 필요성 시험
Self-attention은 순서 정보가 없음 (행렬 연산이라 위치 무관).
→ 위치 정보를 임베딩에 더해줌 = Position Encoding.
"I love you" vs "You love I"가 attention 입장에서는 같음 → 위치 정보 필수
p.20
Position Encoding 방법 3가지
방법특징단점
Absolute각 위치마다 임베딩 학습긴 문장 일반화 약함
Relative단어 간 상대 거리 인코딩계산 복잡
RoPE회전 행렬 적용, 절대+상대 위치-
최신 LLM 대부분은 RoPE 사용
Tokenization & BPE
p.22
Tokenization 개요
문자열을 토큰 시퀀스로 분리.
공백 기반(단어 단위)은 현대 LLM에서 사용 안 함.
p.23
단어 단위 토큰화의 문제
  • OOV: 학습 안 된 단어 처리 불가
  • Subword 정보 손실: "unhappiness" → un+happy+ness 구조 활용 못함
  • 어휘 폭발: 단어 수 = 어휘 크기 → 너무 많아짐
p.24
문자 단위 토큰화의 문제
OOV 없고 어휘 작지만:
  • 시퀀스 길이 폭발 → self-attention은 O(n²) → 느려짐
  • 문자 하나는 의미 없음
p.25
Subword Tokenization 시험
단어 단위 + 문자 단위의 중간점.
알고리즘사용 모델
BPEGPT 계열
WordPieceBERT
SentencePieceT5, LLaMA 등
두 단계: ① Token Learner (어휘 구축) + ② Token Segmenter (분리)
p.26~32
BPE (Byte-Pair Encoding) 상세
직관: 문자 단위에서 시작 → 자주 나오는 쌍을 반복 합치기.
  1. 문자 단위 어휘로 초기화
  2. 가장 자주 나오는 인접 쌍 찾기
  3. 그 쌍을 합쳐서 새 토큰 추가
  4. 코퍼스 업데이트
  5. 지정한 횟수만큼 반복
"er" 빈도 9 → "er_" 빈도 9 → "ne" 빈도 8 ... 순서로 병합
p.33
BPE Token Segmenter
학습된 병합 규칙을 순서대로 적용해서 새 문장 분리.
예: "lower_" → 학습에 없음 → "low" + "er_" 로 분리.
OOV 문제 해결 + subword 의미 활용 가능
추가 Transformer 모듈
p.35
Transformer Block 전체 구조
Multi-head Attention → LayerNorm → FFN → LayerNorm
(각 sublayer마다 Residual connection 적용)
이 블록을 N번 쌓음.
p.36~37
Layer Normalization
학습 중 각 층 입력 분포가 계속 바뀜(Internal Covariate Shift) → 학습 불안정.
LayerNorm(x) = γ × (x - μ)/σ + β
μ, σ: 벡터 내 차원들로 계산. γ, β: 학습 가능 파라미터.
Batch Norm과 달리 벡터 하나 단위로 정규화
p.38
FFN in Transformer
FFN(x) = ReLU(xW₁)W₂
2층 MLP. 매 토큰에 동일 가중치 적용 (토큰 독립).
Self-attention이 "어디에 집중?"이라면, FFN은 "그 정보로 뭘 할까?" 역할.
p.39
Residual Connection 시험
y = x + f(x)
입력을 출력에 그냥 더해줌.
장점:
  • Vanishing gradient 완화 (지름길로 gradient 통과)
  • 정보 흐름 보존
  • 깊은 모델 학습 가능
p.40
Language Model Head
마지막 층 출력 → LM Head → 단어 확률 분포.
Weight tying: 입력 임베딩 행렬 = 출력 행렬 (파라미터 절약 + 성능 향상).
p.41~42
Transformer LM 전체 구조 & 학습
입력 토큰 임베딩 + Position encoding → Transformer 블록 N개 → LM head → 다음 단어 예측.
Cross-Entropy Loss로 학습. 병렬로 모든 위치 동시에 학습 가능 (RNN보다 효율적).
Pretraining
p.44~45
Pretraining 필요성
기존: 각 태스크마다 처음부터 학습 → 데이터 부족, 일반화 약함.
웹에 텍스트 데이터가 엄청 많음 → 이걸로 먼저 학습(Pretraining) 후 Fine-tuning.
p.47
Self-Supervised Learning
정답 레이블 없이도 학습 가능.
입력 일부를 가리거나 손상 → 나머지로 복원/예측.
사람이 레이블링 안 해도 됨.
p.48
Pretraining + Fine-Tuning
  1. Pretraining: 대규모 텍스트로 사전학습 (범용 능력)
  2. Fine-Tuning: 특정 태스크 데이터로 추가 학습 (특화)
Fine-tuning 형태: 분류 레이블 데이터 or 대화 데이터(Instruction tuning).
p.49
Transformer가 Pretraining에 적합한 이유
  • 효율성: 병렬 처리 → 대규모 데이터 빠르게 학습
  • 확장성: 크기 키울수록 성능 향상 (Scaling law)
  • 범용성: 텍스트 외 이미지, 오디오, 멀티모달 적용 가능

BERT Stacking
BERT: Stacking Transformer Layers ⭐ 시험범위 끝
BERT = Transformer Encoder 블록을 여러 층 쌓은 구조.
  • BERT-base: 12층, 768 차원, 12 heads
  • BERT-large: 24층, 1024 차원, 16 heads
입력 토큰 → [CLS]+토큰임베딩+위치임베딩 → Transformer Layer × N → 출력
"The outputs from the last layer become contextual embeddings"
→ 마지막 층의 출력 = 각 토큰의 문맥 반영 임베딩 (Contextual Embeddings).
→ 같은 단어라도 문맥에 따라 다른 벡터를 가짐. (cf. Word2Vec은 항상 같은 벡터)

⚡ 최종 핵심 정리 (암기용)
0~1강 핵심
  1. 언어 모델: 단어 시퀀스에 확률 부여 → 모든 NLP 태스크의 핵심
  2. Autoregressive: P(wₜ | w₁,...,wₜ₋₁)을 순서대로 예측
  3. N-gram 트레이드오프: N↑ → 문맥↑, Sparsity↑, 파라미터↑
  4. Smoothing: Add-one / Add-k / Interpolation / Backoff → 0확률 방지
  5. Perplexity: 낮을수록 좋음, log-scale로 계산
2~3강 핵심
  1. 어휘 관계: Synonyms / Antonyms / Hyponym / Hypernym / Polysemy
  2. WordNet 한계: 수동 구축, 문맥 고려 불가 → Word Embedding 등장
  3. Distributional Hypothesis: 비슷한 문맥 = 비슷한 의미
  4. Sparse vs Dense: Dense = 작은 차원, 비영 실수, 신경망에 효율적
  5. Word2Vec Skip-gram: 중심 단어 → 문맥 단어 예측 + Negative Sampling
  6. Word Analogy: vking - vman + vwoman ≈ vqueen
4강 RNN 핵심 5가지
  1. RNN: 같은 가중치 재사용, hₜ = f(Wₕhₜ₋₁ + Wₓxₜ)
  2. RNN 한계: Vanishing gradient / 장거리 의존성 / 단방향 / Exposure bias
  3. LSTM: cell state + 3 gates (forget / input / output)
  4. Cell state 업데이트: cₜ = fₜ⊙cₜ₋₁ + iₜ⊙c̃ₜ
  5. Bidirectional RNN: 앞→뒤 + 뒤→앞 → concatenate
5강 Transformer 핵심 5가지
  1. Self-Attention: αᵢⱼ = Softmax(qᵢkⱼ/√d), aᵢ = Σαᵢⱼvⱼ
  2. Q/K/V: Q=검색, K=인덱스, V=내용 — 같은 벡터 쓰면 자기한테만 집중
  3. Multi-Head: 여러 head → 다양한 관계 학습 → concatenate
  4. Encoder vs Decoder: BERT=양방향(NLU) / GPT=단방향(NLG)
  5. BERT stacking: 마지막 층 출력 = Contextual Embeddings (문맥 반영)
